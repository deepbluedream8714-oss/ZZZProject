# 데이터 관리 가이드

## 📁 데이터 파일 구조

```
src/data/
├── agents.json          # 에이전트 정보
├── wengines.json       # W-엔진 정보
├── data-schema.json    # 데이터 스키마 정의
└── README.md           # 이 파일
```

## 📝 데이터 수집 방법

### 1. 에이전트 정보 수집

#### 수집 소스
- [호요랩 게임 기록](https://act.hoyolab.com/app/zzz-game-record/)
- 게임 내 캐릭터 정보
- 커뮤니티 빌드 가이드

#### 수집 항목 체크리스트

**기본 정보**
- [ ] ID (고유 번호)
- [ ] 이름 (한글/영문)
- [ ] 등급 (S, A)
- [ ] 속성 (물리, 화염, 얼음, 전기, 에테르, 현묵, 서리열)
- [ ] 소속 (팩션)
- [ ] 특성 (강공, 격파, 이상, 방어, 지원, 명파)
- [ ] 태그 (배열)
- [ ] 이미지 경로

**상세 정보**
- [ ] 스탯 (기본 공격력, 방어력, 체력, 충격력, 최대값)
- [ ] 스킬 (기본, 특수, 궁극기, 패시브)
- [ ] 빌드 추천 (디스크 세트, 무기)
- [ ] 팀 조합 (최적, 대체)
- [ ] 평가 (딜링, 생존력, 지원력, 난이도)
- [ ] 장단점
- [ ] 추천 대상

### 2. W-엔진 정보 수집

#### 수집 항목 체크리스트
- [ ] ID (kebab-case)
- [ ] 이름 (영문)
- [ ] 등급 (S, A, B)
- [ ] 타입 (Attack, Stun, Support, Anomaly, Defence, Rupture)
- [ ] 속성
- [ ] 기본 공격력
- [ ] 부 스탯 (이름, 수치)
- [ ] 설명 (효과)
- [ ] 태그
- [ ] 이미지 경로

## 📋 데이터 입력 형식

### JSON 파일 직접 편집

```json
{
  "id": 1,
  "name": "에이전트 이름",
  "rank": "S",
  "attribute": "얼음",
  "faction": "빅토리아 하우스키핑",
  "specialty": "강공",
  "tags": ["태그1", "태그2"],
  "image": "/src/assets/images/charactor/소속/파일명.png"
}
```

### 스프레드시트 활용

1. Google Sheets 또는 Excel 사용
2. 각 컬럼을 데이터 필드로 매핑
3. JSON으로 내보내기 (스크립트 사용)

## 🛠️ 데이터 관리 도구

### 스크립트 사용

```bash
# 데이터 검증
npm run validate:data

# 데이터 변환 (스프레드시트 → JSON)
npm run convert:data

# 데이터 병합
npm run merge:data
```

## ✅ 데이터 추가 프로세스

1. **정보 수집**
   - 게임 기록 사이트에서 확인
   - 커뮤니티 정보 확인

2. **데이터 입력**
   - `agents.json` 또는 `wengines.json`에 추가
   - 스키마에 맞게 작성

3. **검증**
   - `npm run validate:data` 실행
   - 필수 필드 확인

4. **이미지 추가**
   - 적절한 폴더에 이미지 저장
   - 경로 확인

5. **테스트**
   - 개발 서버 실행
   - 화면에서 확인
